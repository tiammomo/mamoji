# Summarize

Transform long content into concise, actionable summaries while preserving key information.

## When to Use

- Summarizing code changes or diffs
- Condensing documentation
- Creating meeting notes
- Summarizing research findings
- Breaking down long conversations

## Techniques

### For Code
- Focus on what changed and why
- Highlight new files and modifications
- Note any breaking changes
- Identify related files affected

### For Documents
- Extract main points
- Preserve key decisions
- Keep action items
- Maintain context

### For Conversations
- Capture key decisions
- List pending tasks
- Note important context
- Summarize conclusions

## Best Practices

- Use bullet points for readability
- Keep summaries proportional to original length
- Preserve critical details
- Highlight action items or next steps
- Maintain factual accuracy
