# 家庭记账工具 PRD

## 1. 产品概述

### 1.1 产品定位
一款面向家庭的 Web 端记账工具，支持多成员独立账号使用，提供基础记账功能和统计报表分析。

### 1.2 目标用户
- 家庭成员：每个成员拥有独立的登录账号
- 可以查看自己的记账记录，也可以查看家庭共享的账单

### 1.3 产品愿景
让家庭成员养成良好的记账习惯，通过数据分析和统计，了解家庭收支情况，合理规划家庭财务。

### 1.4 MVP 阶段划分

> **当前阶段：MVP (最小可行产品)**
>
> 初期聚焦核心功能，快速迭代验证产品价值。

| 阶段 | 目标 | 时间线 |
|------|------|--------|
| MVP | 前后端基础框架 + 核心记账功能 | 第 1-2 周 |
| V1.0 | 家庭组管理 + 基础统计 | 第 3-4 周 |
| V1.1 | 完善统计报表 + 账户管理 | 第 5-6 周 |
| V2.0 | 高级功能（预算、数据导出等）| 待定 |

### 1.5 MVP 技术架构

> MVP 阶段采用简化架构，后续平滑升级。

| 层级 | MVP 技术选型 | 说明 |
|------|-------------|------|
| 前端 | Next.js 16.x (App Router) | React 18+ |
| 后端 | Spring Boot 3.5.x | Java 21 |
| 数据库 | SQLite (开发) → MySQL (生产) | 开发用 SQLite，生产切换 MySQL |
| 部署 | Docker Compose | 本地开发 / 测试环境 |

> **后续升级路径：**
> - 缓存：可后期集成 Redis
> - 部署：可升级为 K8s

### 2.2 设计模式

#### 2.2.1 前端设计模式

| 模式 | 应用场景 | 实践方式 |
|------|----------|----------|
| Server Components | 页面初始渲染 | 减少客户端 JS 体积 |
| Client Components | 交互组件 | use client 声明 |
| Server Actions | 表单提交 | 渐进增强 |
| Suspense | 异步加载 | 流式 UI |
| Parallel Routes | 并行加载 | 独立加载模块 |
| Intercepting Routes | 模式弹窗 | 拦截导航 |

#### 2.2.2 后端设计模式

| 模式 | 应用场景 | 实现方式 |
|------|----------|----------|
| DDD 分层架构 | 业务逻辑组织 | Controller/Service/Repository |
| Builder 模式 | 复杂对象构建 | Lombok @Builder |
| Specification 模式 | 动态查询 | JPA Specifications |
| Decorator 模式 | 增强服务 | Spring Decorator |
| Event Sourcing | 交易流水 | 事件驱动 |

#### 2.3.3 可用性
| 指标 | 目标值 |
|------|--------|

## 4. MVP 业务流程

### 4.1 注册流程
```
输入邮箱/密码 → 注册成功 → 登录
```

### 4.2 记账流程（核心流程）
```
首页 → 点击记账 → 选择收支类型 → 输入金额 → 选择分类 → 选择日期 → 提交 → 记账成功 → 返回首页
```

### 4.3 查看报表流程
```
首页 → 点击报表 → 查看本月收支 → 查看趋势图/分类图
```
