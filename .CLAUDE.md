# Mamoji - 家庭记账工具

## 项目概述

一款面向家庭的 Web 端记账工具，支持多成员独立账号使用，提供基础记账功能和统计报表分析。

**产品愿景**：让家庭成员养成良好的记账习惯，通过数据分析和统计，了解家庭收支情况，合理规划家庭财务。

## 技术栈

| 层级 | 技术 | 版本 | 说明 |
|------|------|------|------|
| 后端 | Spring Boot | 3.5.x | Java 21 |
| 前端 | Next.js | 16.x | App Router, React 18+ |
| 前端 UI | React + Tailwind CSS | 18.x / 3.4.x | |
| 前端状态 | TanStack Query + Zustand | 5.x / 4.x | 数据请求/缓存 |
| 前端图表 | Recharts | 2.x | 统计图表 |
| 前端图标 | Lucide React | - | 图标库 |
| 数据库 | SQLite (MVP) → MySQL | 8.0 | 开发用 SQLite，生产用 MySQL |
| 缓存 | Redis | 7.x | Session/Token缓存（生产环境） |
| 认证 | JWT (jjwt) | 0.12.x | Token 认证 |
| API | RESTful | - | 基础路径 `/api/v1` |

## 开发规范

### 数据库
- MVP 阶段：SQLite（开箱即用，无需配置）
- 生产环境：MySQL 8.0
- 遵循 DB.md 中的索引设计
- 使用 Redis 7.x 缓存热点数据（用户Session、统计数据）

### API 设计
- 基础路径：`/api/v1`
- 认证方式：JWT Token
- 响应格式参考 API.md 中的通用响应格式

### 文档维护
- 文档放在 `docs/` 目录
- 使用 Markdown 格式
- 参考 [docs-optimizer.md](.claude/docs-optimizer.md) 进行文档优化

## 注意事项

1. **数据安全**：密码需要加密存储，用户敏感信息需加密
2. **家庭隔离**：家庭成员只能访问自己家庭的数据
3. **权限控制**：家庭管理员可管理成员，普通成员只能查看数据
4. **性能要求**：页面加载时间 < 2s，操作步骤不超过 3 步
